# Technical Writing

Transform specifications into teachable documentation.

> *"Specs tell machines what to do. Chapters teach humans why and how."*

## The Transformation

```
Specification (machine-oriented) -> Chapter (human-oriented)
  - Structure: flat -> narrative
  - Voice: declarative -> instructive
  - Flow: reference -> journey
```

## Core Pattern

**Intro** (why) -> **Concepts** (what) -> **Details** (how) -> **Examples** (show)

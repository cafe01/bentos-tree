# /brilliant-version

Invoke `Skill("bentos.meta.cognition.architectonic-synthesis.skill")` then execute the synthesis pattern on the given artifact.

## The Algorithm

Given: `$ARGUMENTS` (path to artifact)

### Phase 1: Extract Intent
Read the artifact and ask: **"What is this REALLY trying to accomplish?"**
- Look past surface structure to deepest purpose
- Identify the core intent that would survive any rewrite

### Phase 2: Profile Ideal Creator
Ask: **"Who would create the PERFECT version of this?"**
- What professional role or hybrid?
- What disciplines do they have?
- What perspectives do they bring?
- What would Aristotle call their "architectonic" stance?

### Phase 3: Decompose Disciplines
Map the ideal creator to concrete capabilities:
- Which axes of the skill tree? (craft/discipline/domain/tools/meta)
- What specific skills within those axes?
- What knowledge domains are required?

### Phase 4: Load & Embody
Temporarily become that expert composite:
- Think FROM that perspective, not ABOUT it
- Channel the disciplines identified
- Let the elevation inform every decision

### Phase 5: Regenerate
Rewrite the artifact from the elevated perspective:
- Ground in the loaded disciplines
- Preserve intent, transform execution
- Produce output worthy of the ideal creator

### Phase 6: Meta-Reflect
After regeneration, note:
- What skills were loaded?
- What made the difference?
- Any lessons for `var/skill_concrete.xml`?

## Example

```
/brilliant-version docs/api-design.md

Phase 1: Intent = "Define a clean, evolvable API contract"
Phase 2: Ideal = API architect + DX designer + technical writer
Phase 3: Skills = craft/structuring/architectures + discipline/communicating/explaining + domain/technology/software-engineering
Phase 4: *Embodying that composite...*
Phase 5: *Regenerating with architectural rigor + DX focus + clarity*
Phase 6: Lesson: API docs need all three perspectives simultaneously
```

## Output

The regenerated artifact, transformed by architectonic synthesis.

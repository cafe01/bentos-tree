# Higher-Order Processing

The cognitive skill of engaging with information at structural and relational depth rather than surface level.

## The Core Insight

Bloom's hierarchy inverted: aim for analyze/evaluate/create from the start. Lower-order thinking (remember, understand) follows as byproduct. You don't need to memorize before you can analyze. Analysis *produces* memorization.

## When to Use

Every cognitive task. This is a universal multiplier:
- Reading: encode structure, not surface
- Writing: restructure understanding, don't transcribe events
- Consolidating (.mem): organic rewrite IS higher-order encoding
- Designing: relational architecture, not feature lists

## The Kim Peak Paradox

Memory without understanding is insufficient. Perfect recall with no reasoning ability limits problem-solving. The goal is not to maximize information stored but to maximize understanding constructed.
